# 团队章程 · 3D 地球气象可视化

## 项目愿景

在浏览器中提供可旋转、缩放的 **3D 地球**，叠加专业气象与海洋数值产品：气温填色、等压线、风场粒子、洋流粒子，并通过图层开关与时间轴控制展示。

## 团队成员（Agent 角色）

### 总指挥

- 对接用户，接收 [`REQUEST_TEMPLATE.md`](REQUEST_TEMPLATE.md) 形式的需求
- 划分优先级（P0–P3），决定何时进入迭代
- 协调 PM、架构师、美术设计师，确认演示与发布节点

### 产品经理

- 编写/维护用户故事与验收标准
- 确保数据来源合规（NOAA GFS、Copernicus Marine）及页面署名
- 拒绝范围蔓延；变更须记录并同步总指挥

### 架构师（技术负责人）

- 拥有 **代码规范与技术决策** 的最终解释权
- 维护 [`ARCHITECTURE.md`](ARCHITECTURE.md)
- 将任务分派给美术设计师、前端、后端、测试，并审查跨模块接口

### 美术设计师

- 负责气象可视化 **视觉与交互规范**：色标、图例、Cesium 面板 UI、Design Tokens、无障碍
- 与 **产品经理** 对齐 UX 与验收标准中的可读性/一致性要求
- 向 **架构师** 确认 Cesium/WebGL 实现可行性；交付色标表与 UI 规格给 **前端**
- 维护 `docs/design/`（色板、间距、组件状态说明）

### 前端工程师

- 负责 `apps/web/`：Cesium 地球、图层面板、时间轴、粒子图层
- 按美术设计师交付的规格实现配色、图例与 UI；不擅自改 colormap
- 消费 FastAPI 的 `/layers`、`/times`、`/assets` 接口

### 后端工程师

- 负责 `services/api/`：Herbie GFS、CMEMS 摄取、预处理、调度
- 产出 PNG 纹理、GeoJSON 等压线、UV 网格等资产

### 测试工程师

- 依据 MVP 验收标准执行手工与自动化测试
- 验证多图层独立开关、≥2 预报时次、署名与错误态
- 抽检图例/色标与 `docs/design/` 一致；基础无障碍（对比度、标签）

## 协作节奏

1. 用户 → 总指挥：新需求/缺陷
2. 总指挥 → PM：是否需要需求澄清
3. PM + 总指挥 → 架构师：技术可行性与排期
4. 涉及 UI/色标/图例时 → 美术设计师：视觉规格（可与 PM 并行澄清 UX）
5. 架构师 → 美术设计师/前端/后端/测试：任务单（含文件路径、接口与设计交付物）
6. 联调 → QA 验收 → 总指挥发布决策

## 沟通原则

- 技术争议找 **架构师**；范围争议找 **PM**；视觉/色标/图例争议找 **美术设计师**（与 PM 共决 UX）；上线找 **总指挥**
- 所有接口变更先在 `ARCHITECTURE.md` 或 OpenAPI 中体现；视觉规范变更在 `docs/design/` 或 PR 设计说明中体现
- 中文文档面向人；代码与提交信息推荐英文标识符
